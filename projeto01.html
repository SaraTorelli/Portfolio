Android
A parte Android do projeto foi desenvolvida exclusivamente para os alunos utilizando o Android Studio. Seu design foi cuidadosamente planejado, começando com uma splash screen que dá as boas-vindas aos usuários, seguida por uma página de login intuitiva. Nessa página, os alunos são solicitados a inserir seu RM (Registro de Matrícula) e senha. Para novos usuários, há um botão de cadastro que permite ao aluno fornecer suas informações, as quais serão validadas em nosso banco de dados. Após a confirmação dos dados, o aluno poderá criar uma nova senha e ter acesso à plataforma. Dentro do aplicativo, o menu inicial oferece uma variedade de recursos essenciais para os alunos. Eles podem acessar suas videoaulas, visualizar suas notas, acompanhar atividades pendentes e consultar o calendário de provas. Além disso, disponibilizamos uma funcionalidade de desempenho que permite aos alunos monitorar seu progresso acadêmico ao longo do tempo. Uma calculadora integrada está disponível para calcular a média final com facilidade, simplificando o acompanhamento do desempenho acadêmico. O menu de navegação (Nav bar) proporciona acesso rápido e fácil a outras seções importantes do aplicativo. Os alunos podem acessar seu perfil, onde encontram informações pessoais e uma foto, que também pode servir como uma carteirinha de estudante virtual. Uma aba de report foi incluída no menu para que os alunos possam relatar dúvidas ou erros na plataforma, garantindo uma comunicação transparente e eficaz com a equipe responsável pelo desenvolvimento. Por fim, uma aba de logout permite aos alunos encerrar sua sessão com segurança quando desejarem. É importante destacar que todo o aplicativo foi desenvolvido em Java, aproveitando ao máximo as capacidades do Android Studio, enquanto o banco de dados foi implementado utilizando o SQL Server, garantindo segurança e eficiência no armazenamento e gerenciamento dos dados dos alunos. Essa abordagem abrangente e cuidadosamente planejada visa não apenas fornecer uma experiência de usuário excepcional, mas também facilitar o acesso e o acompanhamento das atividades acadêmicas, promovendo assim um ambiente de aprendizado mais eficaz e engajador para os alunos.
WEB
A parte web foi desenvolvida para atender às necessidades tanto dos administradores quanto dos professores, utilizando HTML, CSS e JavaScript para garantir uma experiência fluida e interativa. O design foi cuidadosamente elaborado, apresentando uma tela inicial onde o usuário pode selecionar se é um administrador ou professor. No caso de ser o primeiro acesso, é necessário clicar no botão "Não tem cadastro?" e preencher os campos com o CPF e código de contratação. Após esse procedimento, uma nova senha será definida e o login será efetuado com sucesso. Na área destinada aos administradores, é possível encontrar funcionalidades como a consulta de alunos e o cadastro de novos alunos. Este cadastro é enviado diretamente para o banco de dados, o qual está interligado ao aplicativo mobile do aluno, e é verificado durante o processo de login. Além disso , na aba de administradores, é possível verificar e adicionar dados dos professores, garantindo uma gestão completa dos usuários do sistema. Ao acessar a aba de professor, é disponibilizado acesso a diversas funcionalidades, incluindo turmas, alunos, lançamento de notas, atividades, videoaulas e até mesmo a capacidade de adicionar eventos ao calendário, como agendamento de provas e outras atividades importantes. Essa abordagem abrangente visa facilitar o gerenciamento de turmas e melhorar a comunicação entre professores e alunos.
Banco de Dados
O banco de dados foi concebido e implementado utilizando o SQL Server, uma escolha estratégica para garantir eficiência, segurança e escalabilidade. Este banco de dados desempenha um papel fundamental no ecossistema do sistema educacional, pois armazena uma ampla gama de informações cruciais para o funcionamento adequado da plataforma. Em sua arquitetura, o banco de dados é projetado para armazenar e gerenciar dados de diversos tipos de usuários, incluindo administradores, professores e alunos. Para cada tipo de usuário, são definidas tabelas específicas que abrigam informações como dados pessoais, credenciais de acesso e outras informações relevantes para a identificação e autenticação do usuário. Além disso, o banco de dados também é responsável por armazenar dados relacionados ao desempenho acadêmico dos alunos, como notas e atividades. Cada interação dos alunos com a plataforma, como submissão de atividades e resultados de avaliações, é registrada e armazenada de forma organizada e segura. Um aspecto crucial do banco de dados é sua capacidade de verificar e validar os dados inseridos. Isso é fundamental para garantir a integridade e a precisão das informações armazenadas. Por meio de procedimentos armazenados, gatilhos e outras técnicas de validação de dados, o banco de dados é capaz de garantir que apenas informações válidas e consistentes sejam inseridas e manipuladas no sistema. Além disso, o banco de dados desempenha um papel ativo na interação entre os diferentes componentes do sistema. Ele fornece serviços de autenticação e autorização, garantindo que apenas usuários autorizados tenham acesso aos recursos e funcionalidades apropriados. Além disso, o banco de dados também é responsável por alimentar informações em tempo real para outras partes do sistema, como a interface do usuário e os aplicativos móveis. Em suma, o banco de dados SQL Server é o alicerce fundamental sobre o qual o sistema educacional é construído. Sua capacidade de armazenar, gerenciar, verificar e validar uma variedade de informações essenciais desempenha um papel crítico na entrega de uma experiência educacional robusta, eficiente e segura para todos os usuários envolvidos.
